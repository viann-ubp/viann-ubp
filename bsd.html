Post 1: Harapan Saya dari Kuliah Basis Data

Saya ingin mata kuliah Basis Data membantu saya memahami cara penyimpanan dan pengelolaan data dengan teratur. Hampir semua aplikasi, dari e-commerce, media sosial, hingga sistem akademik, pasti bergantung pada database.

Setelah mempelajarinya, saya berharap bisa menguasai konsep dasar, membuat desain database sendiri, dan lebih mahir menulis query SQL. Harapannya, ilmu ini bisa bermanfaat di kampus maupun di dunia kerja.

Post 2: Apa Itu Basis Data?

Basis data adalah tempat penyimpanan data agar tetap rapi dan mudah diakses. Bisa diibaratkan seperti rak arsip digital, hanya saja pencarian data di database jauh lebih cepat dengan bantuan komputer.

Contoh penggunaannya sangat banyak: data akun media sosial, transaksi belanja online, hingga daftar mahasiswa di kampus. Database biasanya dikelola oleh DBMS seperti MySQL, PostgreSQL, atau Oracle. Dengan kata lain, database adalah pusat informasi sekaligus "otak" aplikasi modern.

Post 3: Apa Itu Conceptual DB?

Sebelum membuat database nyata, kita memulainya dengan gambaran abstrak bernama Conceptual Database. Model ini mendefinisikan entitas, atribut, dan relasi antar entitas.

Contohnya dalam sistem kampus: ada entitas Mahasiswa, Dosen, dan Mata Kuliah. Hubungannya bisa berupa “Mahasiswa mengambil Mata Kuliah” atau “Dosen mengajar Mata Kuliah”. Biasanya divisualisasikan dalam bentuk ERD (Entity Relationship Diagram) agar alur data lebih jelas.

Post 4: Apa Itu Physical Model?

Setelah model konseptual selesai, tahap berikutnya adalah Physical Model. Pada tahap ini, database diwujudkan ke dalam tabel, kolom, tipe data, serta penentuan primary key dan foreign key.

Misalnya, entitas Mahasiswa menjadi tabel dengan kolom NIM, Nama, dan Alamat. Entitas Mata Kuliah menjadi tabel lain, lalu ada tabel tambahan untuk data KRS. Inilah yang nantinya diimplementasikan di MySQL atau PostgreSQL.
